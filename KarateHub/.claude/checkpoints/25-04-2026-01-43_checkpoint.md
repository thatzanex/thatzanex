## Matchup Guide Generation Protocol

### Mathematical Framework for Guide Authoring

When asked to write a guide for **Style A vs Style B**, derive strategy using this exact process:

#### 1. Range Dominance
```
rangeWinner = argmax(max attack_range_studs across all moves)
```
Longer range → kiting/poke strategy. Shorter range → close-gap / pressure strategy.

#### 2. Speed Interruption
```
Compare delay_seconds for each shared move.
If A.delay < B.delay → A can interrupt B's startup.
```
The style with lower `delay_seconds` has the "interrupt" tactic.

#### 3. Mobility Kiting
```
kiteExploit = A.move.walk_speed > B.fastestChaseSpeed
```
If a style can move backward while attacking faster than the opponent can chase,
they have a **mathematical kiting exploit** — cite exact numbers.

#### 4. Block Trade Analysis
```
guardBreakValue = guard_break_duration_seconds (penalty if guard broken)
postBlockSpeed  = post_block_attack_delay_seconds (how fast can you punish)
```
Higher `guard_break_duration_seconds` = riskier to block incorrectly.
Lower `post_block_attack_delay_seconds` = faster parry punish.
